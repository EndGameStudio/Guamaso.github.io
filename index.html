Wakanda Data Import

Wakanda Data Import Tool

When new data is created to support the application, not user data, we need a way to update servers down the deployment chain. With SQL, this was easy… ish. A versioned query that was part of the deployment handled the task. With Wakanda, we could do the same thing. OR we could take a step higher.

I was tasked with creating a bare bones rights engine for our app. In it, I had to display error messages to the user. I don’t like hardcoding messages, I prefer to let the client edit them whenever possible. I created the solution for this but then ran into a problem syncing data across development and test servers - which meant we were going to run into a problem syncing the data to production… once we reached that point.

So, I created a Datastore class method that checked for a JSON file and imported any required data into the target datastore. It worked great. A couple of issues did come up like exposing the method to the client and abstracting the schema of the table from the import process.

Having the method exposed to the client is dangerous for many reasons, most importantly being that our team is still new to Wakanda, which is also still in its infancy. The solution is to be able to call the method from the server itself. Or easily disable the method once the app is in “production” mode.

The schema part I’m still debating on how to handle. I can include the schema in the import file. I could verify the schema based on the key/value pairs in the data itself. The main task is complete and was approved so the next time I have to do something similar, I’ll go in and clean it up.
