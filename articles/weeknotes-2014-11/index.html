Consolidated sites to Digital Ocean

Digital Ocean doesn’t have redundancies and, so far, they seem to have issues more often than not. It is marketed as a developer VPS solution, which, to me, says stability is not to be expected. And you know what? That’s ok. It’s good enough and it is insanely cheap. The sites that I put on there aren’t popular even by any definition, so if they go down even for a few hours here and there, it should be fine. If they grow in popularity (ha! Right…) then it will make more sense to move them over to something else. Maybe AWS or A Small Orange (they have a similar plan at $20 but with redundant hardware, for example).

Digital Ocean has a ton of great guides for installing all sorts of things. Ghost 0.5 was just released so they are missing the updated version number, but the process is the same. I learned how to have multiple Ghost instances running under one NginX server and how to configure them all as services. I love their “droplet” interface as well. Even with having to use command line for the server itself, their interface is so much easier to use then cPanel (I was told that you can install a GUI and use that to admin the server, but I’m starting to like command line!). Here’s a referral sign up link if you’re interested, which you should be. :)

Explored Laddr

Laddr is nifty PHP app that helps Code for America Brigades keep track of their projects, conversations, and other stuff. It was created as a Code for America project (I think?) in Philadelphia. As I said before, I’m helping out with the Cary Brigade and the first area I’m helping out with is the front end to the Brigade’s website. Laddr is commonly placed on the root domain for many Brigades. We’re not set up for that now but we might in the near future.

I’m curious how Laddr and Github will mesh as far as communication on bugs and features. The administration page is nifty. I can edit the PHP templating and other code easily through this back end page. I’ll get to really dig in this weekend and I’m super exited.
