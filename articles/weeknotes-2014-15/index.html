Last Day

Finished my last day at KSS. Good times there. It’s always sad to end a relationship. I still would like to maintain the individual relationships I made and even the organizational relationship as well. What I’m trying to say is that the relationship that I had between myself and the employer is done and that makes me feel a little bummed out. You spend all this time building relationships with people and the work itself and then bam, it’s done. It’s easy to maintain a connection when you have to be at a place doing a specific list of tasks. Without that constraint, keeping those relationships is difficult.

On the other hand, the end of one chapter leads to the start of another. New people, new challenges, new puzzles. Can’t wait.

Scraping with Cheerio

The Town of Cary maintains a historic theater named The Cary. The town also maintains it’s website that lists all the show times. Apparently, they don’t do a great job of using the data to it’s fullest potential. The Code for America brigade that I help out at wants to parse out that data and create a front end experience that is more helpful to potential customers.

I started with figuring out how to scrape a page with Cheerio. It’s amazingly easier than I thought it would be. The problem is with the structure of the page. As I’m building the scraping module, I’m hoping that they don’t change the structure very often, if at all. Without unique attributes to target specific elements, I have to use the structure to find the show time listings on the page.

The show times themselves have a very volatile structure. So, I’m trying to use tools like Moment to figure out what cell has what piece of data. That’s where I am at right now. All of this is in a prototyping stage but I also want to start storing the raw scrapes somewhere… somewhere being MongoDB. The more data points I have, the more accurate my parser will be.

MongoDB

There are so many technologies out there. NoSQL data storage tools alone are a handful. I just got a new machine and I’ve decided that instead of installing every tool I want to learn or use on the computer itself, I will spin up a VirtualBox machine and install it on there instead. It’s a little bit of an overhead as far as installing and using go but the ability to completely turn off a service is worth it to me.

MongoDB, for example, I know will not be taking up system resources when the VirtualBox machine is turned off. Let’s say I need to install MySQL or something similar. I can do the same thing for those. Development of the application still happens on my machine, they just point to the correct IP/Port to access the service.

Having never used Mongo, I’m looking over the documentation on how to do basic save/edits to data. Found a tool called Mongoose for NodeJS.
