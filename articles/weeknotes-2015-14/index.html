StealJS yet again

I spent some time working on the build process for The Art Project. It seems like i was using the incorrect distribution of CanJS. Once i switched to AMD, things started working. Im still having problems though. Now the stache plugin is returning some funky errors. Still debugging what the errors actually mean.

A question i have to bring up is why Steal doesn’t report better errors. Ill explain later as I’m still trying to make sure I m on the right track.

Promises and deferreds

I was working on a dialog implementation using can.component. Example usage of the component’s api was given to me and I just had to build it. Im glad i was given these examples as it seriously open my mind to the real possibilities of deferreds.

Being able to use reject and resolve to trigger different callback without having to pass the callbacks to the dialog itself is genius. Ill be adding this to our component library asap.

Web Workers

I read a primer on web workers this week and I’m curious if they can be used to compile templates. I am using a pseudo model to manage a calendar widget, I bet this can be refactored to be a web worker as well.

It’s crazy how much the browser api and ECMA script combined have essentially become a programming environment like native mobile apps. Its cool and exciting, but also a little crazy.
